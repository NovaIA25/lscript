# Frontend vs Backend : quelle différence ?

Quand on parle de développement web, on entend souvent "frontend" et "backend". Mais c'est quoi exactement, et quelle est la différence ?

## Frontend = ce que tu vois

Le **frontend**, c'est tout ce que l'utilisateur voit et touche :

- Les couleurs du site
- Les boutons
- Les textes
- Les animations
- Le menu de navigation

Quand tu visites un site, tout ce qui s'affiche sur ton écran, c'est du frontend.

**Technologies utilisées :** HTML, CSS, JavaScript

## Backend = ce que tu ne vois pas

Le **backend**, c'est tout ce qui se passe "derrière" :

- Vérifier ton mot de passe quand tu te connectes
- Enregistrer tes données dans une base de données
- Envoyer des emails
- Calculer le prix de ton panier

Tu ne le vois jamais, mais c'est lui qui fait fonctionner l'application.

**Technologies utilisées :** Python, Node.js, PHP, bases de données

## Un exemple concret

Tu cliques sur "Se connecter" avec ton email et mot de passe.

| Étape                        | Qui fait quoi                      |
| ---------------------------- | ---------------------------------- |
| Tu tapes ton email           | **Frontend** affiche le formulaire |
| Tu cliques sur "Valider"     | **Frontend** envoie les données    |
| Vérification du mot de passe | **Backend** compare avec la base   |
| Réponse "OK" ou "Erreur"     | **Backend** répond                 |
| Affichage du message         | **Frontend** montre le résultat    |

## Deux métiers différents

- **Développeur frontend** : se concentre sur l'interface utilisateur
- **Développeur backend** : se concentre sur la logique et les données
- **Développeur fullstack** : fait les deux

## Par où commencer ?

Si tu débutes, commence par le **frontend** (HTML, CSS, JavaScript). C'est plus visuel et plus gratifiant au début.

Le backend viendra naturellement quand tu voudras ajouter des fonctionnalités plus avancées.
